解决方案

为什么“解决方案”在五步法里面很重要？(需求 -> 解决方案 -> 商业模式 -> 增长 -> 壁垒)
- 可以低成本测试核心单元
- 把某一个重要的价值因素从单元模型里抽出来，提前测一遍 


产品内核：用户愿意选择你的最小解决方案
- 用户需求。产品内核 就是在探究用户的价值点强弱，要站在用户需求视角想。
- 愿意选择。【用户价值 =（新体验-旧体验）- 替换成本】 替换阻力
- 最小。早期产品要做糙一点；如果做多了，判断就模糊了，无法知道哪个是产品内核的核心价值。
- 解决方案。用户真正关心的问题，而不是功能清单。服务特性 vs 功能清单

内核三原则
- 工具：分析价值的战略取舍工具
- 分层：逐步使用。
    - 分清哪些是内核，哪些是优化。
- 精准：不大不小刚好

内核修炼-六个段位
1. 缺价值意识
2. 有价值意识
3. 有最小意识
4. 有专业评估
5. 有快速迭代
6. 有迁移能力


产品内核就是提出的假设，验证假设通过：常识 < 调研 << 测试 <<< 全量开发

通常在早期： 留存 >> 获客，价值 >> 效率，成立 >> 复制
